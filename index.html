GRAPHS: THE BASICS

What is a graph?
A graph is a mathematical structure that models relationships between objects. Each object is called a node or vertex, and the connections between them are called edges. Graphs are used everywhere: social networks, maps, recommendation engines, network routing, and much more.

Key Components:

  • Node (Vertex): A point, representing an entity (like a city or a person).
  • Edge: A connection between two nodes (like a road between cities, or a friendship between people).

Visual Representation

In the diagram above:

- The lines are edges connecting the nodes.

Types of Graphs

  • Undirected: Connections go both ways, like a two-way street. Example: Facebook friends.
  • Directed: Connections have a direction, like a one-way street. Example: Twitter followers.
  • Weighted: Edges have values (weights), such as road distances or costs.
  • Unweighted: All connections are equal; no special value associated.

Graphs are the foundation of many algorithms. Let’s explore how we use them!

GRAPH ALGORITHMS

Breadth-First Search (BFS)

BFS is a classic algorithm for exploring graphs. It starts at a node and explores all its neighbors before moving further. Imagine ripples spreading in water—BFS visits nodes level-by-level.
How BFS works:

  1. Start at the source node
  2. Mark it as visited, add to a queue
  3. Dequeue node, visit all its unvisited neighbors, mark them visited, and enqueue them
  4. Repeat until the queue is empty
Uses: Finding the shortest path in unweighted graphs, social networks, web crawlers, and more.

Depth-First Search (DFS)

DFS explores as deep as possible along each branch before backtracking. It’s like navigating a maze—going as far as you can before hitting a wall.
How DFS works:

  1. Start at the chosen node
  2. Mark as visited
  3. Recursively visit each unvisited neighbor
  4. Backtrack when needed
Uses: Puzzle solving, network analysis, finding connected components.

SHORTEST PATH IN GRAPHS

The shortest path between two nodes is the path with the fewest edges (or lowest total weight in weighted graphs). In unweighted graphs, BFS efficiently finds the shortest path.

Step-by-step:

  1. Start at the source node
  2. Explore neighbors layer by layer
  3. Track how you got to each node
  4. Once you reach the target node, trace back your steps
Uses: Routing, network optimization, urban planning.

Dijkstra's Algorithm

Dijkstra's Algorithm is a famous algorithm for finding the shortest path in weighted graphs. Unlike BFS, Dijkstra’s can handle edge weights and always finds the least-cost path.
How Dijkstra's Algorithm works:

  1. Start with the source node, set distance to 0, all others to infinity
  2. Put all nodes in a priority queue (min-heap) based on current shortest distance
  3. Pick the node with the smallest distance, update distances to its neighbors if a shorter path is found
  4. Repeat until all nodes are visited or the target node’s shortest path is found
Applications: GPS navigation, network routing, resource allocation.
Pseudocode: 
function dijkstra(graph, source):
    dist[source] = 0
    for each node in graph:
        if node != source:
            dist[node] = infinity
    while unvisited nodes remain:
        current = node with smallest dist
        for neighbor of current:
            if dist[neighbor] > dist[current] + edge_weight:
                dist[neighbor] = dist[current] + edge_weight
        mark current as visited
        if current is target: break
